# 2026-07-12 工作日志

## octopus-game/ — BUG 修复（4 项）

### BUG1: 图片宽高比变形
- 问题：draw() 中所有图片画成正方形 `size × size`，shark（1022×662）和 jellyfish（524×824）变形
- 修复：Entity 构造函数记录 `imgRatio = img.width / img.height`，draw() 中按比例计算 drawW/drawH

### BUG2: 点击屏幕开始游戏
- 问题：游戏加载后自动开始，无开始界面
- 修复：新增 #startScreen overlay（标题+规则+点击提示），加载完成后显示开始界面，点击后 startGame()

### BUG3: 玩家只控制鲨鱼 + 限时挑战
- 移除 1/2/3 切换逻辑和 switchPlayer() 函数
- 鲨鱼始终 isPlayer=true，speed 从 1.6 提升到 3.5
- 新增 60 秒倒计时，右上角显示，最后 10 秒红色闪烁
- 游戏结束条件从"角色被吃"改为"时间到"
- endGame 面板标题改为"时间到！"

### BUG4: 游动逻辑细化 + 数量翻倍
- **鱼群抽搐修复**：wander() 移除每帧随机扰动 `+= (random-0.5)*0.1`，改为仅在 wanderTimer 到期时小幅转向（±0.3 弧度，间隔 90~180 帧）
- **游出屏幕**：AI 实体移除边界反弹，允许游出；玩家鲨鱼保留边界 clamp
- **屏幕外重生**：完全离开屏幕（3 倍 radius 外）后从随机边缘外侧重生，速度朝向屏幕内
- **数量翻倍**：章鱼 1→2，水母 1→2，鱼群 20→40
- 新增 respawnFromEdge() 方法处理屏幕外重生

## octopus-game/ — 第三轮改进（5 项）

### 1. 去掉时间限制
- 移除 60 秒倒计时、timer HUD、timer CSS、updateTimerDisplay()、gameLoop 中的倒计时逻辑
- 游戏改为纯无尽模式，无结束条件
- R 键随时重新开始（不再需要 gameOver 状态）
- 移除 gameOver 面板相关代码

### 2. 速度层级：鲨鱼 > 鱼群 > 章鱼/水母
- 新增 SPEED_BASE = 5.0 常量
- 鲨鱼: 5.0 (1.0x), 鱼群: 4.0 (0.8x), 章鱼/水母: 3.0 (0.6x)
- ENTITY_TYPES 中 speed 改为 SPEED_BASE * 系数

### 3. 所有物体不能重叠
- 新增 resolveOverlaps() 函数：遍历所有存活实体对，非捕食关系时检测重叠并推开
- 分离距离 = (a.radius + b.radius) * 1.15
- 玩家不被推开，只推开对方；非玩家双方各退一半
- gameLoop 中在 checkCollisions() 之前调用

### 4. 鱼群左右方向变少频率
- wanderTimer 从 90~180 帧（1.5~3秒）改为 300~600 帧（5~10秒）
- 角度偏移从 ±0.3 弧度减小为 ±0.2（基于 baseDirection）
- 仅 10% 概率反转方向，保持水平游动为主
- 构造函数中鱼群初始 wanderTimer 也改为 300~600

### 5. 数量调整
- 鱼群 40→30，章鱼 2→3，水母 保持 2

## octopus-game/ — 第四轮改进（4 项）

- 章鱼/水母: 3.0 (0.6x) 不变

### 3. 鱼群方向变化保持 5 秒间隔
- wanderTimer 保持 300~600 帧（5~10 秒）
- 角度偏移 ±0.4 弧度（基于 baseDirection）
- 10% 概率反转左右方向
- 大部分时间沿固定方向平稳游动

### 4. AI 行为调整

- **鱼群** 不再躲避捕食者，仅 wander 平滑游动
## octopus-game/ — 第五轮改进（4 项）

### 1. 积分只算鲨鱼
- `eatEntity()` 中移除章鱼/水母吃鱼群的加分逻辑
- 只有鲨鱼吃章鱼、水母、鱼时才会加分和弹出 `+score` 提示
- 章鱼/水母仍会吃小鱼，但不再增加分数
- 更新 HUD 与开始界面规则文案，说明“章鱼/水母吃小鱼不算得分”

### 2. 章鱼/水母感知与逃逸优化
- 感知鲨鱼范围从 220px 减半到 110px
- 逃逸速度从 `sp * 1.2` 减半到 `sp * 0.6`
- 逃逸方向不再直接设置 velocity，而是使用 `vx = vx*0.85 + target*0.15` 插值平滑，避免左右摆动
- 整体逃逸更平稳

### 3. 鲨鱼体积变大 2.5 倍
- `ENTITY_TYPES.shark.radius` 从 40 提升到 100
- 鲨鱼绘制尺寸从约 100px 扩大到 250px，视觉体积明显变大
- 碰撞/进食范围同步扩大

### 4. 鱼群朝向修复
- 鱼图 `Yu_1.png` 默认朝左，原代码 `vx < 0 时 flip` 导致鱼群看起来倒着游
- 改为鱼群单独逻辑：`vx > 0 时翻转`，使其始终面朝游动方向
- 其他精灵（鲨鱼、章鱼、水母）保持 `vx < 0 时翻转`（假设默认朝右）
